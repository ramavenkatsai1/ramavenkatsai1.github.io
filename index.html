Projects

Dynamic Student Survey Form, Department Course Catalog

Angular, AJAX, HTML, CSS, Python, Flask, SQL, SQL Server, Docker, Kubernetes, AWS (S3, EC2, Elastic Beanstalk), Kafka, WordPress

Project Overview

This project revolved around creating a dynamic student survey form and a department course catalog. It was aimed at improving the digital transformation efforts of the university department, making course selection and feedback seamless for students.

Steps and Implementation

  1. Frontend Development with Angular: Designed a user-friendly interface for the survey form and course catalog. Utilized Angular components, directives, and services to make the platform dynamic and responsive. Incorporated AJAX to asynchronously fetch data, enhancing the user experience by eliminating full-page reloads.
  2. Styling with HTML & CSS: Leveraged HTML5 and CSS3 features to make the interface visually appealing and intuitive. Ensured that the layout was mobile-responsive and accessible across different devices.
  3. Backend with Flask and Python: Flask, a lightweight Python web framework, was chosen for its simplicity and extensibility. Developed RESTful routes, integrated with Angular via REST APIs, and managed session data for user authentication and course selections.
  4. Database Integration using SQL Server: Used SQL Server as the primary database to store course data, student information, and survey results. Designed an efficient database schema, implemented stored procedures, and managed data transactions with Flask's ORM layer.
  5. Integrating WordPress: To make content management easier for department administrators, integrated WordPress modules into the platform. This allowed for easy updates to course information, news, and other relevant content without the need for technical intervention.
  6. Containerization with Docker: To ensure consistency across development, staging, and production environments, the application was containerized using Docker. This made sure that the app ran uniformly, regardless of where it was deployed.
  7. Kubernetes for Orchestration: With the aim of ensuring high availability and managing containers at scale, Kubernetes was used. It managed the deployment, scaling, and networking of the application, especially beneficial when deploying to cloud environments.
  8. Deployment on AWS: Leveraged AWS services like S3 for storing static assets, EC2 for running the application instances, and Elastic Beanstalk for orchestration. AWS provided a robust and scalable environment to handle growing user demands.
  9. Streaming with Kafka: To handle real-time data flow and ensure that the system could manage large influxes of survey responses, integrated Kafka. It acted as a message broker, efficiently managing communication between different parts of the application.
  10. Challenges faced:
    • Integration with WordPress modules required a deep understanding of WP internals.
    • Scaling the application using Kubernetes presented a learning curve, especially ensuring that the services communicated seamlessly within the cluster.
    • Performance tuning of the SQL Server to handle concurrent student accesses during peak times.

Real-time Travel Planning Application

React, Node, TypeScript, Python, Apache, Docker, AWS, REST API, OpenWeatherMap API, SIEBEL CRM

Project Overview

This project aimed to develop a robust real-time travel planning application, facilitating users to create, modify, and share travel itineraries dynamically. The platform provided live weather updates and map functionalities, making travel planning efficient and interactive.

Steps and Implementation

  1. Frontend Development with React: Utilized React to build a user-centric and interactive interface. Enhanced the application's performance with efficient component rendering and state management, ensuring a seamless user experience.
  2. Backend Implementation using Node and TypeScript: Chose Node.js for its non-blocking I/O capabilities and TypeScript for static typing, ensuring a robust backend. This combination facilitated faster response times and enhanced code reliability.
  3. Python for Business Logic: Python's versatile libraries and easy-to-read syntax made it ideal for implementing the core business logic of the application. From fetching real-time data to handling user interactions, Python streamlined the backend operations.
  4. Integration with SIEBEL CRM: Integrated SIEBEL CRM to manage user data, feedback, and interactions, ensuring that customer relations were always at the forefront. This CRM tool helped in capturing user requirements, feedback, and resolving any concerns rapidly.
  5. Microservices Architecture: By adopting a microservices architecture, the application was broken down into smaller, independent services. This not only improved scalability and fault isolation but also facilitated easier updates without affecting other parts of the system.
  6. Apache Server Hosting: Chose Apache servers for hosting due to their reliability and performance. Apache provided a stable environment for the application, managing requests and serving content efficiently.
  7. API Integration: Integrated OpenWeatherMap API to fetch live weather updates and REST APIs for data exchange between the frontend and backend. This provided users with real-time insights into their planned destinations, enhancing their travel experience.
  8. Docker for Containerization: Used Docker to containerize the application, ensuring consistency across different environments. This made deployments faster, scalable, and more predictable.
  9. Deployment using AWS EKS: AWS Elastic Kubernetes Service (EKS) was used to orchestrate the Docker containers. This made the scaling, deployment, and management of the application efficient on the AWS cloud infrastructure.
  10. Challenges faced:
    • Integrating multiple data sources like OpenWeatherMap and ensuring real-time data accuracy presented a challenge.
    • Scaling the application to handle peak user loads without latency was crucial and required fine-tuning of resources.
    • Ensuring data synchronization between the application and SIEBEL CRM required meticulous planning and testing.
